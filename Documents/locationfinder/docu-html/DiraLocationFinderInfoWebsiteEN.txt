<p class=MsoNormal><b><span lang=EN-AU style='font-size:12.0pt;line-height:

<p class=MsoNormal><b><span lang=EN-AU style='font-size:12.0pt;line-height:
107%;font-family:"Fedra Sans";color:#215E99;mso-themecolor:text2;mso-themetint:
191'>Architecture<o:p></o:p></span></b></p>

<p class=MsoNormal><span lang=EN-AU>The LocationFinder is a server-side
solution consisting of two programs: </span></p>

<p class=MsoListParagraphCxSpFirst style='text-indent:-18.0pt;mso-list:l1 level1 lfo6'><![if !supportLists]><span
lang=EN-AU style='mso-bidi-font-family:Aptos;mso-bidi-theme-font:minor-latin'><span
style='mso-list:Ignore'>1)<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span></span><![endif]><span lang=EN-AU>a <span class=GramE>Builder</span>
and </span></p>

<p class=MsoListParagraphCxSpLast style='text-indent:-18.0pt;mso-list:l1 level1 lfo6'><![if !supportLists]><span
lang=EN-AU style='mso-bidi-font-family:Aptos;mso-bidi-theme-font:minor-latin'><span
style='mso-list:Ignore'>2)<span style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span></span></span><![endif]><span lang=EN-AU>a Service. </span></p>

<p class=MsoNormal><span lang=EN-AU>The Builder reads location data and
constructs an index structure on disk. The Service uses this index structure to
respond to queries directed to it via a simple REST API. (The Service also
implements a subset of the Esri REST API to allow integration into the ArcGIS
platform.) The Builder reads data in a simple XML format. (With the help of
plugins, it can also read Esri feature classes and some other formats.) The
build configuration determines how the search  feels  (behaves) to the
end-user. For example, it is possible to generate keywords for categories to
assist the user in searching, shoot in synonyms, and to some extent, determine
orthographic tolerance. Furthermore, it is possible to derive aggregates from
existing locations, such as streets and postal code areas from address points.</span></p>

<p class=MsoNormal><span lang=EN-AU style='font-size:12.0pt;line-height:107%'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><b><span lang=EN-AU style='font-size:12.0pt;line-height:
107%;font-family:"Fedra Sans";color:#215E99;mso-themecolor:text2;mso-themetint:
191'>Integration into your application(s)<o:p></o:p></span></b></p>

<p class=MsoNormal><span lang=EN-AU>The Builder program has a graphical
interface for creating the build configuration. However, the Service program
only provides a REST API and no interface for end-users. The Service is
designed to be integrated into an application (or into the ArcGIS platform).
Dira GeoSystems has extensive experience in integrating the LocationFinder into
your applications.</span></p>

<p class=MsoNormal><span lang=EN-AU><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><b><span lang=EN-AU style='font-family:"Fedra Sans";
color:#215E99;mso-themecolor:text2;mso-themetint:191'>System Requirements<o:p></o:p></span></b></p>

<p class=MsoNormal><span lang=EN-AU>The LocationFinder requires .NET and a fast
local disk for the index. There are no additional dependencies, making
installation and operation very simple.</span></p>

<p class=MsoNormal><span lang=EN-AU>The latest version of the Service runs on
Windows and on Linux, including inside a Linux-based Container. The Builder is
a Windows application.</span></p>
